# 展覽介紹稿與分工企劃 (修訂版 - 融入評分準則)

## 1. 總結 (Summary)
為四名六年級學生分配展覽介紹角色，並撰寫一份簡單易懂、具吸引力的 Markdown 格式介紹稿。本次講稿將**自然且流暢地融入比賽的六大評分準則**（定義問題、設計及方法、執行、創意、潛在影響、表達技巧），讓學生在講述「大姐姐大哥哥計劃」的發現、3D打印、AI訓練及遊戲設計時，不知不覺間回答評判心中的所有疑問。同時會為每位學生標示「重點記住的地方」，確保他們能團隊合作，也能獨立應答。

## 2. 現前狀態分析與準則對應 (Current State Analysis & Rubrics Mapping)
- **評分準則要求**：
  - **I. 定義問題 (20%)**：說明問題是什麼、為何重要、幫助誰。
  - **II. 設計及方法 (15%)**：計劃過程、參考過往做法、創新與差異性。
  - **III. 執行 (15%)**：按部就班執行、測試與收集數據、成功與否。
  - **IV. 創意 (25%)**：原創性、獨特地結合不同技術。
  - **V. 潛在影響 (15%)**：幫助有需要的人、解決真實問題。
  - **VI. 表達技巧 (10%)**：清晰、易懂、自信作答。

- **專案背景**：結合 Micro:bit（硬體體感）、AI 動作辨識、網頁藍牙（Web Bluetooth）與 React 遊戲介面（「行行出狀元」主題）。

## 3. 建議改動與角色分配 (Proposed Changes & Role Assignment)
為了讓介紹流暢不生硬，四位同學將化身為一個「產品開發團隊」，每人負責一個核心環節，並自然帶出相應的評分準則：

- **同學 A (隊長 / 產品發起人)** 
  - **對應準則**：I. 定義問題、V. 潛在影響
  - **負責內容**：開場。講述在「大姐姐大哥哥計劃」中觀察到的痛點（初小/非華語學生覺得寫中文字很悶、難記）。說明這個問題的真實性與重要性，以及我們的作品希望能讓學寫字變得輕鬆有趣，對社區產生正面影響。
  
- **同學 B (創意總監 / 遊戲設計師)**
  - **對應準則**：II. 設計及方法、IV. 創意
  - **負責內容**：講述我們參考了傳統的抄寫簿（過去做法），但決定跳出框框。介紹「具身認知」（用身體記住筆劃）的創新想法，以及我們如何將 Micro:bit 體感技術與「行行出狀元」的網頁遊戲獨特地結合在一起。

- **同學 C (AI 數據工程師)**
  - **對應準則**：III. 執行、IV. 創意
  - **負責內容**：講述 AI 動作訓練的過程。我們如何一步步收集「橫、豎、撇、捺」的揮動數據，進行測試並改良模型。分享實驗過程中的挑戰與最終的成功結果。

- **同學 D (硬體工程師)**
  - **對應準則**：II. 設計及方法、III. 執行
  - **負責內容**：介紹 3D 打印外殼的設計計劃與原型製作。說明如何透過不斷測試，設計出適合小朋友手握、保護 Micro:bit 且方便揮動的外殼，展示我們確切執行的成果。

## 4. 講稿結構設計 (Script Structure)
使用 `Write` 工具在 `.trae/documents/presentation_script.md` 生成講稿。結構如下：
1. **團隊介紹與開場**（同學 A）
2. **創新設計與玩法**（同學 B）
3. **AI 訓練與測試**（同學 C）
4. **硬體設計與改良**（同學 D）
5. **總結與影響**（同學 A）
6. **附錄：獨當一面 Q&A 秘笈**（為每人列出應對評判提問的核心要點，對應「VI. 表達技巧」）。

## 5. 驗證步驟 (Verification Steps)
1. 檢查講稿是否以六年級學生的口吻撰寫（自信、清晰、不幼稚）。
2. 檢視講稿內容是否**隱性但明確地**覆蓋了圖片中 I 至 V 的所有問題點。
3. 確認每位學生的重點提示是否能幫助他們自信應答。
