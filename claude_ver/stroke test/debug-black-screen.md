# [OPEN] debug-black-screen

## 問題摘要

- 症狀：Vite 頁面可打開，但使用者回報畫面全黑。
- 預期：首頁或應用主畫面應正常渲染，而不是空白/黑屏。

## 目前假設

1. 路由或根組件在首屏渲染時發生執行期錯誤，導致 React 畫面中斷。
2. 新增的鼓勵模式狀態或 props 在 `GeminiApp.jsx` / `main.jsx` 初始化時讀取了空值，造成首屏 crash。
3. CSS 疊層或全屏 overlay 常駐顯示，實際不是 crash，而是被黑色遮罩蓋住。
4. 瀏覽器 console 存在未捕獲錯誤，但 build 階段無法發現，因此需要在執行期收集證據。
5. dev server 雖已啟動，但目前打開的路由與預期不同，進入了某個會渲染黑底調試頁或異常狀態頁。

## 證據收集計畫

- 檢查 dev server 當前狀態與路由輸出。
- 用瀏覽器快照與 console 訊息確認是 crash、遮罩還是路由問題。
- 如有必要，再加入最小化執行期插樁。

## 狀態

- 已建立調試會話，等待收集第一輪執行期證據。
